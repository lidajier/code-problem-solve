## 题目 1：二维数组最小路径和

### 题目描述

给定一个二维数组 `grid`，从左上角 `(0, 0)` 出发，走到右下角 `(m - 1, n - 1)`。

每次只能：

- 向右走一格
- 向下走一格

要求：

- 把路径上经过的所有数字加起来
- 求这个和的最小值

例如：

```text
1 3 1
1 5 1
4 2 1
```

答案是：`7`

---

### 难度判断

- 这题通常算经典动态规划题
- 难度大致是简单偏中等，或者基础 Medium
- 如果平时刷过题，会觉得它不难
- 如果是第一次接触动态规划，会觉得有点绕

它主要考：

- 能不能识别出动态规划
- 会不会定义状态
- 能不能写出状态转移方程
- 会不会处理第一行、第一列这种边界情况

---

### 核心思路

先不要急着背公式，先理解一句话：

> 走到当前格子的最小代价 = 走到前一个格子的较小代价 + 当前格子的值

为什么？

因为走到某个位置 `(i, j)` 时，你最后一步只能来自两个地方：

- 上面 `(i - 1, j)`
- 左边 `(i, j - 1)`

所以，到 `(i, j)` 的最小路径和，一定是：

- 从上面走过来的最小路径和
- 和从左边走过来的最小路径和

这两者里面更小的那个，再加上当前格子的值。

---

### dp 数组的含义

定义：

```cpp
dp[i][j]
```

表示：

> 从左上角走到位置 `(i, j)` 时，路径和的最小值。

这句话必须记住。

因为只要这句话理解了，后面的转移方程就能顺着推出来。

---

### 状态转移方程

```cpp
dp[i][j] = min(dp[i - 1][j], dp[i][j - 1]) + grid[i][j]
```

解释：

- `dp[i - 1][j]` 表示从上边走到当前位置
- `dp[i][j - 1]` 表示从左边走到当前位置
- `min(...)` 表示选更便宜的那条路
- `+ grid[i][j]` 表示加上当前格子的值

---

### 为什么第一行和第一列要单独处理

因为第一行没有上边，第一列没有左边。

所以：

#### 起点

```cpp
dp[0][0] = grid[0][0]
```

#### 第一行

第一行只能一直从左往右走：

```cpp
dp[0][j] = dp[0][j - 1] + grid[0][j]
```

#### 第一列

第一列只能一直从上往下走：

```cpp
dp[i][0] = dp[i - 1][0] + grid[i][0]
```

---

### 手动推一遍例子

原数组：

```text
1 3 1
1 5 1
4 2 1
```

#### 第一步：起点

```cpp
dp[0][0] = 1
```

#### 第二步：第一行

```cpp
dp[0][1] = 1 + 3 = 4
dp[0][2] = 4 + 1 = 5
```

第一行变成：

```text
1 4 5
```

#### 第三步：第一列

```cpp
dp[1][0] = 1 + 1 = 2
dp[2][0] = 2 + 4 = 6
```

现在表格是：

```text
1 4 5
2 ? ?
6 ? ?
```

#### 第四步：处理中间格子

`(1, 1)` 的值是 `5`：

- 上面是 `4`
- 左边是 `2`
- 取更小的 `2`

```cpp
dp[1][1] = min(4, 2) + 5 = 7
```

`(1, 2)` 的值是 `1`：

- 上面是 `5`
- 左边是 `7`
- 取更小的 `5`

```cpp
dp[1][2] = min(5, 7) + 1 = 6
```

`(2, 1)` 的值是 `2`：

- 上面是 `7`
- 左边是 `6`
- 取更小的 `6`

```cpp
dp[2][1] = min(7, 6) + 2 = 8
```

`(2, 2)` 的值是 `1`：

- 上面是 `6`
- 左边是 `8`
- 取更小的 `6`

```cpp
dp[2][2] = min(6, 8) + 1 = 7
```

最终 `dp` 表：

```text
1 4 5
2 7 6
6 8 7
```

所以答案就是右下角：`7`

---

### C++ 代码（详细注释版）

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

int minPathSum(vector<vector<int>>& grid) {
    // m 表示行数
    int m = grid.size();

    // n 表示列数
    int n = grid[0].size();

    // 创建 dp 数组
    // dp[i][j] 表示从左上角走到 (i, j) 的最小路径和
    vector<vector<int>> dp(m, vector<int>(n, 0));

    // 起点，只能是自己
    dp[0][0] = grid[0][0];

    // 处理第一行，因为第一行只能从左边过来
    for (int j = 1; j < n; j++) {
        dp[0][j] = dp[0][j - 1] + grid[0][j];
    }

    // 处理第一列，因为第一列只能从上边过来
    for (int i = 1; i < m; i++) {
        dp[i][0] = dp[i - 1][0] + grid[i][0];
    }

    // 处理其余位置
    for (int i = 1; i < m; i++) {
        for (int j = 1; j < n; j++) {
            dp[i][j] = min(dp[i - 1][j], dp[i][j - 1]) + grid[i][j];
        }
    }

    // 返回右下角结果
    return dp[m - 1][n - 1];
}

int main() {
    vector<vector<int>> grid = {
        {1, 3, 1},
        {1, 5, 1},
        {4, 2, 1}
    };

    cout << "最小路径和: " << minPathSum(grid) << endl;
    return 0;
}
```

---

### 最容易出错的地方

1. 把题目方向记错，标准题一般是向右和向下
2. 忘记初始化第一行
3. 忘记初始化第一列
4. 没先想清楚 `dp[i][j]` 的含义，就直接硬套公式

---

### 这题一句话总结

> 每个格子都只需要比较两条路：从上边来，还是从左边来，哪个更小就选哪个。
